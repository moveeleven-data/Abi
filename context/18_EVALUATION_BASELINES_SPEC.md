# Phase 11: Evaluation, Baselines, and Human-Trace Import v1 Spec

## Status

Phase 11 implements the first evaluation and baseline-comparison packet.

This phase does not finalize an artifact.

This phase does not claim real human validation.

This phase does not run a live human study.

## Goal

Create an evaluation packet around a candidate artifact produced by the source-to-artifact production run.

The evaluation packet should represent:

candidate artifact
→ baseline artifacts
→ blind comparison
→ reader-state transition comparison
→ human-trace import or fixture trace
→ baseline comparison report
→ evaluation gate report
→ evaluation packet

The purpose is to make comparison and reader-state evidence first-class before any final artifact or paper claim.

## Inputs

Default inputs should be deterministic fixtures and the latest fake production packet.

Optional explicit input path may be supported later, but tests must use fixtures only.

## Required CLI behavior

Add a command such as:

abi evaluation demo --client fake

and optionally:

abi evaluation demo --client openai --allow-live-model --max-model-calls 12

Rules:

- fake client mode must not require API key
- openai client mode must require --allow-live-model
- openai client mode must require OPENAI_API_KEY
- deterministic demos must remain unchanged
- evaluation packet output must use unique packet directories
- accepted parsed model artifacts must include model_call_id where model-shaped outputs are used
- all artifacts must be registered through the artifact registry
- finalization must remain fail-closed

## Required artifacts

The evaluation packet should produce artifact types equivalent to:

- evaluation_subject
- evaluation_candidate_artifact_ref
- evaluation_direct_prompt_baseline
- evaluation_best_of_n_baseline_summary
- evaluation_blind_comparison_protocol
- evaluation_blind_comparison_result
- evaluation_human_trace_import
- evaluation_reader_state_transition_comparison
- evaluation_baseline_comparison_report
- evaluation_gate_report
- evaluation_packet

Exact artifact names may differ if aligned with existing conventions.

## Candidate and baseline rules

The candidate artifact must remain marked:

- non_final: true
- not_human_validated: true
- not_finalization_eligible: true

Baselines must be clearly marked as fixture/fake unless generated by an explicit guarded live path.

No baseline result may be treated as real external validation in this phase.

## Human trace rules

Phase 11 may import a fixture human trace from fixtures/human_calibration/ or create a new deterministic fixture trace.

The trace must be marked:

- fixture_only: true
- not_real_validation: true

No live survey integration.

No real human-study claim.

## Model-call behavior

The fake evaluation run may use fake model-shaped workers for baseline summaries or comparison reports.

Every model-shaped output must be schema-validated before parsed artifact registration.

Accepted parsed model artifacts must include:

- schema_version
- artifact_type
- run_id
- lineage_id
- parent_ids
- created_by
- fixture_only
- model_call_id
- payload

Invalid model outputs must not create parsed artifacts.

Client failures must be recorded.

## Budget guard

Add or reuse a simple max-call guard.

Suggested default:

--max-model-calls 12

If execution would exceed the budget, the command must refuse or stop with a structured error.

## Required gate behavior

Phase 11 gate report evaluates only the evaluation packet scaffold.

It must not claim that Abi has beaten baselines.

It must not claim a phase shift.

It must not satisfy finalization gates.

Finalization must remain fail-closed.

## Prohibited

- no final artifact claim
- no phase-shift claim
- no real human-study claim
- no live human-study automation
- no dashboard
- no multi-lineage tournament
- no agent loops
- no automatic OpenAI calls
- no finalization semantic changes
- no SKILL.md
- no broad refactor

## Acceptance criteria

Phase 11 is complete only when:

- ruff passes
- pytest passes
- deterministic demos still work
- fake evaluation demo works
- openai evaluation demo refuses without --allow-live-model
- openai evaluation demo refuses without OPENAI_API_KEY
- evaluation packet contains candidate, baselines, blind comparison, human-trace fixture/import, comparison report, gate report
- fixture-only and not-real-validation flags are present where required
- candidate remains non-final
- parent IDs are populated where appropriate
- max-call budget is enforced
- finalization remains fail-closed
