# Raw-Model Baseline Protocol

Last checked: 2026-06-18

## Definition

The raw-model baseline is generated by a direct model prompt without Abi packet orchestration, Abi Ear, Minimal Reread, production harness composition, hidden consequence scaffolding, or operator selection informed by Abi internals.

## Prompt Rules

The raw prompt must include the same source material and target constraints available to Abi. It may ask for a text artifact but must not include Abi-specific vocabulary such as self-isomorphic reread, hidden consequence graph, opening transfiguration, or Abi Ear moves.

The raw prompt must be frozen before output generation. Model, provider, configuration, timestamp, and model-call record must be stored.

## Budget Rules

The raw-model baseline must receive a fair budget. If Abi uses multiple model calls in a future live run, the raw baseline must receive a preregistered comparison budget, such as one direct call plus a best-of-N variant set. The choice must be justified before seeing reader results.

## Output Handling

No manual polishing is allowed unless a matching polishing rule is available to all arms. If multiple raw outputs are generated, the selection rule must be locked before scoring.

## Gate Relationship

Future evidence may support `raw_model_baseline_comparison_passed` only after blinded reader traces show the Abi candidate meets the preregistered superiority or non-inferiority threshold against the raw baseline. This document does not pass the gate.
